Key Design Decisions

üîß Architecture

Modular AI Agent Pattern

  • Telegram Bot Interface ‚Üí n8n Workflow ‚Üí AI Agent Orchestration
  • Separate specialized tools (Image Analyzer, Search, Think Tool)
  • Prevents database logging for non-product messages

ü§ñ AI Model Choice

Google Gemini Flash 2.5

  • Fast inference (<1sec for conversations)
  • Multi-modal: voice transcription + image vision API
  • Long context window (handles extended conversations)
  • Cost-effective for high-volume requests

üíæ Database Strategy

Dual-Table Approach

  • n8n_chat_histories: Conversation memory for context
  • Prevents image loss with separate Get Images Link DB using Google Sheet
  • Query ID tracking maintains session state

üß† Memory System

Window Buffer (Supabase-backed)

  • Stores last 10-20 messages per user (by chat_id)
  • Eliminates repetitive questions
  • Maintains conversation context across multiple inputs
  • Enables smart follow-up without re-querying

‚öñÔ∏è Risk Scoring

Weighted Multi-Factor Algorithm

  • Price discount: 40% weight (>40% off = red flag)
  • Seller authorization: 25% weight
  • Image quality: 20% weight (<6/10 = concerning)
  • Packaging authenticity: 15% weight
  • Batch code format: 10% weight
  • Output: 0.0-1.0 score ‚Üí Verdict

üîç Image Analysis

Forensic Vision Approach

  • Detects: print quality, typography, logo proportions
  • Returns 0-10 quality score + specific red flags
  • Separate database storage (prevents loss on follow-up)

üõ†Ô∏è Tool Orchestration

Conditional Triggering

  • Image Analyzer: Always runs on photo upload
  • Search Tool: Triggered if price >30% off or seller unknown
  • Think Tool: Mandatory for every product message
  • Database Logger: SKIPPED for greetings

üîÑ Conversation Flow

State Machine

  • Intake ‚Üí DataCollection ‚Üí Research
  • Analysis (Think) ‚Üí Logging ‚Üí Response
  • Loop or End based on user interaction
